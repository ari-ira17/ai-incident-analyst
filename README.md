# 🏢 AI-аналитик инцидентов

**Решение кейса в рамках конкурса "Цифровой прорыв"**

Автоматизированная система анализа, классификации и обработки инцидентов в сфере жилищно-коммунального хозяйства (ЖКХ) и муниципальных услуг с использованием искусственного интеллекта.

## 📋 Описание

Приложение предназначено для ускорения и повышения качества обработки жалоб граждан на проблемы в сфере ЖКХ. Система использует LLM (Large Language Models) через Ollama для:

- **Анализа** данных об инцидентах и генерации аналитических отчётов
- **Классификации** жалоб по категориям и маршрутизации в нужные отделы
- **Генерации** проектов ответов на основе контекста инцидента

## 🚀 Основные возможности

### 📊 Модуль "Аналитик"
- Загрузка и обработка данных об инцидентах в формате Excel
- Два режима обработки: быстрое решение (fast) и AI-решение (slow)
- Автоматический расчёт критичности проблем
- Генерация рейтингов муниципалитетов по тяжести проблем
- Визуализация распределения проблем по регионам (диаграммы Plotly)
- Экспорт отчётов в формате TXT (от Ollama AI) и XLSX

### 🤖 Модуль "Классификатор"
- Многоэтапная LLM-классификация инцидентов:
  - **Этап 1**: Определение is_problem (проблема или не проблема)
  - **Этап 2**: Классификация по тематикам
  - **Этап 3**: Маршрутизация в соответствующий отдел
- Пакетная обработка с отслеживанием прогресса
- Поддержка гибкой структуры входных данных
- Экспорт результатов в CSV

### ✍️ Модуль "Генератор"
- Автоматическая генерация проектов ответов на жалобы
- Учёт контекста и категории инцидента
- Очистка текстов от спама и артефактов (VK-ссылки, URL, HTML-теги, иероглифы)
- Сохранение результатов для редактирования
- Экспорт в CSV для дальнейшей обработки

## 📦 Требования

- Python 3.9+
- [Ollama](https://ollama.ai/) (для работы LLM) — требуется локальная установка
- Virtual Environment (venv)

## 🔧 Установка

### 1. Клонирование репозитория
```bash
git clone https://github.com/ari-ira17/ai-incident-analyst
cd ai-incident-analyst
```

### 2. Создание виртуального окружения
```bash
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Запуск приложения
```bash
streamlit run app.py
```

Приложение откроется в браузере на `http://localhost:8501`

### Рабочий процесс

1. **Главная страница** — ознакомление с возможностями системы
2. **Аналитик** — загрузите файл с инцидентами, выберите режим обработки, получите отчёт
3. **Классификатор** — автоматическая классификация инцидентов по темам и отделам
4. **Генератор** — создание проектов ответов на жалобы граждан

### Форматы данных

**Входной формат**: Excel файлы (.xlsx)

Минимальные требования:
- Колонка `Текст инцидента` — содержание жалобы/описание проблемы

Опциональные колонки:
- `Муниципалитет` — название муниципалитета
- `Отдел` — ответственный отдел
- `Тема` — категория проблемы
- `Тип инцидента` — тип инцидента

**Выходные форматы**:
- CSV с результатами классификации и генерации
- XLSX с рейтингами муниципалитетов
- TXT отчёты от AI

## 📊 Примеры использования

### Быстрый старт с тестовыми данными

1. Откройте приложение: `streamlit run app.py`
2. Перейдите на страницу "Аналитик"
3. Включите опцию "Использовать тестовые файлы"
4. Выберите режим ("Базовое решение" или "AI-решение")
5. Нажмите "Запустить анализ"

### Загрузка своих данных

1. На странице "Аналитик" или "Классификатор" отключите "Использовать тестовые файлы"
2. Загрузите свой Excel файл
3. Нажмите кнопку запуска
4. Скачайте результаты
